# 결과 요약 개선 명령서

결과 요약 문구를 "분석 절차 설명"이 아니라 "핵심 위험 요약" 중심으로 수정해줘.

## 현재 문제

결과 화면의 큰 제목에 표시되는 `task_summary`가 "RULA 분석을 수행했다", "작업 자세 분석을 통해..." 같은 시스템 절차 설명으로 생성되고 있다.

사용자가 알고 싶은 것은 분석을 했다는 사실이 아니라, 이 작업에서 실제로 어떤 위험이 발견되었는지다.

## 수정 목표

1. `task_summary` 또는 새 필드 `risk_summary`가 분석 절차 설명을 하지 않도록 한다.
2. 최종 점수, 위험 수준, 고위험 구간 수, 주요 위험 부위, 주요 자세 요인, 집중 시간대를 포함한 핵심 위험 요약을 생성한다.
3. LLM 자유 생성에만 맡기지 말고, 백엔드에서 계산 가능한 요약 재료를 만든 뒤 LLM이 문장화하도록 한다.
4. UI의 가장 큰 제목에는 절차 설명이 아니라 핵심 위험 요약을 표시한다.

## 권장 구현 방향

- 가능하면 응답 스키마에 `risk_summary`, `risk_highlights`를 추가한다.
- 기존 호환성이 필요하면 `task_summary`는 유지하되 의미를 "핵심 위험 요약"으로 바꾼다.
- UI는 다음 우선순위로 표시한다.

```jsx
<h2>{llm.risk_summary || llm.task_summary}</h2>
```

## 백엔드 수정

`evidence_bundle` 또는 LLM 입력 생성 단계에서 `computed_summary`를 추가한다.

포함할 값:

```json
{
  "computed_summary": {
    "final_score": 7,
    "final_action": "즉각적인 개선 필요",
    "high_risk_window_count": 4,
    "main_body_parts": ["몸통", "목", "손목"],
    "main_drivers": ["몸통 굴곡", "몸통 비틀림", "목 굴곡", "손목 부담"],
    "high_risk_times": ["00:06 - 00:09", "00:13 - 00:14", "00:16 - 00:18", "00:29 - 00:31"]
  }
}
```

주의:

- 영어 키값이 있다면 사용자 표시용으로 한글 변환한다.
- 예: `trunk` -> `몸통`, `neck` -> `목`, `wrist` -> `손목`, `upper_arm` -> `상완`, `lower_arm` -> `전완`, `leg` -> `다리`.

## 프롬프트 수정

`SYSTEM_PROMPT` 또는 LLM 지시문에 아래 규칙을 추가한다.

```text
task_summary 또는 risk_summary에는 분석 절차나 시스템 동작을 설명하지 마라.
예: "RULA 작업 자세 분석을 통해...", "평가를 수행했습니다" 같은 문장은 금지한다.

요약문에는 반드시 다음 중 3개 이상을 포함하라.
- 최종 점수 또는 위험 수준
- 고위험 구간 수
- 주요 위험 부위
- 주요 자세 요인
- 위험이 집중된 시간 구간

사용자가 바로 행동 판단을 할 수 있는 핵심 위험 요약을 작성하라.
```

## 권장 출력 형식

```json
{
  "risk_summary": "최종 보정 점수는 7점으로 즉각적인 개선이 필요한 수준이며, 52초 작업 중 4개의 고위험 구간이 확인되었습니다. 주요 위험은 몸통 굴곡과 비틀림, 목 굴곡, 손목 부담이 반복적으로 함께 나타난 점입니다. 특히 6~9초, 13~18초, 29~31초 구간에서 고위험 자세가 집중되었습니다.",
  "risk_highlights": [
    "최종 보정 점수 7점으로 즉각적인 개선 필요",
    "고위험 구간 4개 확인",
    "몸통, 목, 손목 부담이 주요 특징"
  ]
}
```

## Verifier 또는 fallback 처리

요약문에 아래 표현이 포함되면 경고하거나 백엔드 계산값 기반 fallback 요약으로 대체한다.

금지 표현:

```text
분석을 통해
평가를 수행했습니다
식별하고
기반으로 인간공학적 평가
작업 자세 분석을 통해
```

Fallback 요약 예:

```text
최종 보정 점수는 {final_score}점으로 {final_action} 수준이며, 전체 작업 중 {high_risk_window_count}개의 고위험 구간이 확인되었습니다. 주요 위험 부위는 {main_body_parts}이고, 주요 자세 요인은 {main_drivers}입니다. 위험은 {high_risk_times} 구간에 집중되었습니다.
```

## UI 수정

1. 결과 화면의 큰 제목은 `risk_summary || task_summary`를 표시한다.
2. 제목 아래에는 `risk_highlights`를 bullet 형태로 표시한다.
3. `risk_highlights`가 없으면 백엔드 계산 요약값으로 다음 항목을 표시한다.

예:

```text
- 최종 보정 점수 7점
- 고위험 구간 4개
- 주요 부위: 몸통, 목, 손목
```

## 검증

- 샘플 JSON을 업로드했을 때 큰 제목이 "분석을 수행했습니다" 같은 절차 설명으로 나오지 않아야 한다.
- 제목에는 실제 위험 점수, 위험 구간 수, 주요 부위 또는 시간대가 포함되어야 한다.
- 기존 `task_summary`만 사용하는 응답도 화면이 깨지지 않아야 한다.
- 금지 표현이 포함된 LLM 응답은 fallback 요약으로 대체되어야 한다.
